# Minecraft RAG AI 챗봇 개발 히스토리 (Part 1: 기반 구축 및 데이터 수집)

## 1. 프로젝트 개요 및 초기 환경 설정
* **목표:** 마인크래프트 위키 데이터를 기반으로 정확한 게임 정보를 제공하는 지능형 RAG(Retrieval-Augmented Generation) 챗봇 개발.
* **기술 스택:** Python, LangChain, ChromaDB (벡터 DB), Google Gemini API, Streamlit (웹 UI), HuggingFace Embeddings (`jhgan/ko-sroberta-multitask`).
* **초기 트러블슈팅:**
  * `langchain_community` 등의 패키지를 설치했음에도 `ModuleNotFoundError`가 발생하는 문제 직면.
  * 원인: `.zshrc` 파일에 `python` 명령어가 시스템 파이썬으로 강제 연결(alias)되어 있어 가상환경(venv)이 무시되는 현상 발견.
  * 해결: alias를 제거하고 ` ./venv/bin/python`으로 명시적 실행하여 환경 분리 완수.

## 2. LLM 연동 및 API 모델 탐색
* **이슈:** Google Gemini API 호출 시 `404 NOT_FOUND` 에러 발생 (`gemini-pro`, `gemini-1.5-flash` 모델 인식 불가).
* **해결 과정:**
  * `langchain-google-genai` 패키지 버전 충돌을 의심하여 여러 버전으로 재설치 테스트 진행.
  * `google.generativeai.list_models()` 코드를 직접 실행하여 발급받은 API 키에서 실제로 접근 가능한 모델 목록을 조회.
  * 해당 API 키 환경에서는 최신 모델인 **`gemini-2.5-flash`**가 활성화되어 있음을 확인하고, 코드를 이에 맞춰 수정하여 성공적으로 LLM 연동 완료.

## 3. 데이터 품질(Quality) 고도화: 3x3 조합법 파서 개발
* **이슈:** 초기 텍스트 크롤링만으로는 마인크래프트의 핵심인 '3x3 제작대 조합법'을 챗봇이 제대로 설명하지 못함.
* **해결:**
  * 단순 텍스트 수집을 넘어, HTML 구조(`mcui-Crafting_Table`)를 분석하는 정밀 파서(Parser)를 `BeautifulSoup`으로 직접 구현.
  * 웹페이지의 이미지 아이콘 속성(title)을 추출하여 `[1번칸: 참나무 판자]` 형태의 텍스트 좌표 데이터로 변환하는 로직 완성.
  * "데이터의 양을 늘리기 전에 질을 먼저 높여야 한다"는 원칙 하에 고품질 데이터 수집 기반 마련.

## 4. 데이터 양(Quantity) 확장: 공식 위키 자동화 수집 (`batch_loader_full.py`)
* **구현:**
  * 마인크래프트 공식 위키 API(`action=query`)를 활용한 대량 문서 수집 스크립트 작성.
  * `processed_items.txt`를 도입하여 중단 후 재시작 시 중복 수집을 방지하는 이어받기(Resume) 로직 구현.
  * 카테고리 내 하위 카테고리까지 파고드는 **재귀 탐색(Recursive Search)** 적용.
* **수집 결과:**
  * 아이템, 몹, 생물 군계, 구조물, 마법 부여, 레드스톤, 명령어, 패치 노트(버전 역사) 등 핵심 공식 문서 약 886개 수집 완료.

## 5. 커뮤니티 팁 및 글리치 데이터 보완 (`namuwiki_loader.py`)
* **이슈:** 공식 위키에는 유저들이 실전에서 쓰는 꿀팁, 스피드런 기술, 버그성 플레이(글리치) 정보가 부족함.
* **해결:**
  * 나무위키의 마인크래프트 팁 문서들을 타겟으로 하는 별도의 심층 크롤러(Depth=1) 제작.
  * 탐험, 파밍, 회로, PVP, 스피드런 등 알짜배기 하위 문서 수집.
  * 공식 데이터와 섞여 환각(Hallucination)이 발생하는 것을 막기 위해 메타데이터에 `{"source": "namuwiki"}` 꼬리표 부착.
  * 약 2,231개의 커뮤니티 팁 데이터 청크 추가 확보.

## 6. 프롬프트 엔지니어링 (뇌 구조 고도화)
* `app.py`의 프롬프트를 대폭 수정하여 챗봇의 답변 퀄리티 향상.
* **주요 가이드라인 적용:**
  * 마크다운을 활용한 가독성 높은 포맷팅.
  * 몹, 구조물, 조합법 등 질문 유형에 따른 맞춤형 답변 지시.
  * **면책 조항 강제:** 공식 위키에 없는 커뮤니티 팁/글리치를 답변할 때는 반드시 `[⚠️ 주의: 이 내용은 공식 위키에 없는 커뮤니티 팁/버그이며...]` 라는 경고문을 서두에 달도록 설정하여 환각 문제 방어.

## 7. 현재 직면한 한계 (Next Step)
* **기억 상실증 (Context Amnesia):**
  * 챗봇이 바로 이전 대화의 문맥(주어)을 기억하지 못함.
  * 예: "워든"에 대해 대화하다가 "잡는 방법이 뭐야?"라고 물으면, 주어인 워든을 잊어버리고 DB에서 단순히 "잡는 방법"과 연관된 엉뚱한 문서(근거지 찾는 법 등)를 가져와 대답하는 현상 발생.
* **해결 과제:** 대화 기록을 인지하여 질문을 재구성하는 **'기록 인지 검색기(History-Aware Retriever)'** 도입 필요.
